CONNECTIONS

渠道与访问方式

每个渠道都讲清楚“从哪里连接、认证在哪里完成、数据最终保存在哪里”。微信、邮箱和手机网页不是同一种连接方式。

微信 · 和 PA 沟通等待连接

微信是日常发送指令和接收结果的主要入口。连接过程在 PA 桌面端展示一次性二维码，不要求用户理解内部会话。

1桌面端生成一次性二维码，并保持页面打开。
2用微信扫码并确认；PA 检测成功后显示已连接身份。
方式：客户端二维码
PA 邮箱 · 给 Agent 发资料可接收

公网邮箱身份随 Cloud 资源授权获得；真实邮件通过用户配置的转发或自管 MTA 投递到 Node，本项目不托管 SMTP/IMAP，正文和附件只存本机。

1获得地址：agent@chen-node.private.site。
2配置邮件提供商转发或本机 MTA，并发送测试邮件。
结果：本机只读收件箱
手机网页与 Online Pages在线

手机查看依赖安全穿透。PA 活动、Worker、邮件和私有页面走受保护入口；公开 Online Page 可使用托管域名或自定义域名。

托管域名 · 推荐点击后在默认浏览器完成 Cloud 授权；返回客户端后读取已分配域名并启动穿透，手机打开域名即可进入 PA。
自定义域名在默认浏览器填写域名 → 复制 `_personal-agent` TXT 验证记录 → 等待 DNS 验证 → 验证通过后绑定 Pages。
当前：chen-node.private.site
