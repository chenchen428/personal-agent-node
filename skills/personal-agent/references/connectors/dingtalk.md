# 钉钉

使用企业内部应用机器人的 Client ID 和 Client Secret 建立钉钉官方 Stream 连接。先读取脱敏状态；配置和清空只允许从本机控制台发起。每个应用只能绑定一个 Space，不输出凭据、Access Token 或会话 Webhook。

## 能做什么

- 从钉钉单聊或群聊继续主 Agent 对话
- 接收文字、图片、文件、语音、视频和富文本；媒体会下载到当前 Space 并登记为受管文件
- 回复文字、图片、安全文件、任务进展和最终结果
- 使用官方 Stream 长连接并自动恢复

钉钉会按会话类型限制媒体能力：人与机器人单聊可接收文件、语音和视频；群聊 @机器人时，钉钉官方可能不投递这些类型。单个接收附件上限 50 MB；原生回发媒体受钉钉上传接口限制为 10 MB。超出限制或钉钉不支持的场景必须明确报告失败，不得退化为本机路径、裸文件链接或未治理的外链。

## 配置 SOP

1. 在钉钉开放平台创建企业内部应用并添加机器人。
2. 为机器人选择 Stream 模式，完成应用版本发布。
3. 在本机连接页面填写 Client ID 和 Client Secret；Personal Agent 先向钉钉校验凭据，再保存到当前 Space 的本机密钥目录。
4. 等待状态变为“已连接”，先发送文字验证会话，再在人与机器人单聊中发送一张图片和一个小于 10 MB 的安全文件完成媒体收发验证。

同一个钉钉应用只能绑定一个 Space。清空配置会停止当前 Space 的 Stream 连接、删除本机应用凭据并释放安装级独占绑定；本机对话记录和用户文件保留。

## CLI 交互

命令入口：`pa-cli connection dingtalk`

CLI 只提供 `status` 脱敏状态读取。Client Secret 不接受命令行参数，配置与清空均在本机连接页面完成。

| 操作 | 风险 | 说明 |
| --- | --- | --- |
| `status` | R0 | 读取脱敏连接状态、最近连接时间和最近收信时间。 |
