# Sites

检查本地站点与穿透状态，生成平台域名使用计划并在确认后应用；不读取或改写站点承载的用户内容。

## 能做什么

- 访问本地站点
- 查看站点健康状态
- 使用平台域名
- 建立和恢复安全穿透

## CLI 交互

命令入口：`pa-cli connection sites`

提供 list、status、use-platform-domain、remove-platform-domain 和 tunnel-status；域名与穿透变更需要确认。

| 操作 | 风险 | 说明 |
| --- | --- | --- |
| `list` | R0 | 列出本地站点。 |
| `status` | R0 | 检查本机访问和穿透状态。 |
| `use-platform-domain` | R2 | 展开横向五步 SOP，在 3 分钟总时限内完成平台授权、域名与穿透分配、标准验证 Page 发布、公网内容标记回读和状态提交；只有最终域名返回 HTTPS 200 且标记一致才写入绑定成功。 |
| `remove-platform-domain` | R2 | 移除本机的平台域名与穿透绑定并确认状态回调；保留 Workspace 数据和平台登记。 |
| `tunnel-status` | R0 | 读取脱敏穿透健康状态。 |

验证 Page 必须通过正式 Page 发布合约一次生成标题、简介、完整 HTML、1200×750 桌面缩略图、750×1200 移动缩略图和 manifest，缺少任一产物不得进入公网检测。页面使用温暖编辑式博客设计，介绍 Personal Agent Node 的本地能力、发布边界、验证原因和适用场景；缩略图与正文共用同一视觉层级。

运行时会持续检查每个节点。验证成功后提供“查看验证发布”和同一公网地址的手机二维码。授权节点必须明确提示用户在浏览器核对 Owner 账号并确认，收到平台回调后才能进入下一节点。
