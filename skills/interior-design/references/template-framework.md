# 装修设计交付页模板框架

以 Personal Agent Pages 内置的“装修设计交付页模板”为实现参照。模板只固定高质量交付所必需的框架，不固定设计答案。

## 固定框架

- 默认展示完整户型的完成态 3D 概念模型，首屏可辨识空间关系与主要家具。
- 提供 `3D 鸟瞰`、`平面`、`室内`三类视角，以及重置到完整户型的操作。
- 使用 OrbitControls 支持用户主动旋转、平移和缩放；没有自动巡游、播放、时间轴或入场动画。
- 空间浏览默认是“整体方案”。选择房间后相机真正进入该空间近景，不用地面高亮代替空间切换。
- Web 使用完整浏览布局；移动端默认横屏，空间选择位于 Header，视角工具保持紧凑并避免遮挡主体。
- 竖屏时给出非阻断的横屏建议，仍允许继续查看。
- 不包含用户编辑器。用户通过自然语言让 Agent 修改模型与页面。
- WebGL 不可用时展示同一模型的可拖拽 3D 投影降级，不退化为空白或静态平面图。

## Agent 自由发挥

根据户型证据和用户意图自由决定：

- 房间数量、空间几何、连通关系和命名；
- 家具品类、尺寸、布局、软装和绿植；
- 墙地材质、配色、灯光、风格与视觉细节；
- 项目标题、概念说明、方案亮点和重点空间；
- 适合当前户型的默认镜头与房间近景距离。

这些细节不得硬编码成示例户型。只有交互框架、设备适配、概念精度说明和安全边界是稳定约束。
