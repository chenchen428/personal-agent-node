# Renovation Project Brief

Use this structure for whole-home and multi-room work. Omit empty narration; keep unknowns explicit.

## 1. Project Snapshot

```text
Project / jurisdiction / property type / building age / occupancy during work
Measured area / drawing status / target completion / budget boundary / current stage
```

Record whether dimensions are site-measured, taken from an approved drawing, user-reported, or estimated. Name the drawing scale and revision when known.

## 2. Household And Use

Capture:

- household members, regular visitors, future changes, mobility and accessibility needs;
- weekday and weekend routines, work-from-home, cooking, laundry, bathing, hobbies, hosting, pets, and storage growth;
- temperature, daylight, noise, privacy, cleaning, repair, and indoor-air concerns;
- three outcomes that matter most and three outcomes the user wants to avoid.

Translate style labels into observable choices. “原木奶油风” is not a specification; record desired contrast, wood tone, gloss, texture, light quality, visual density, and tolerance for patina or maintenance.

## 3. Existing Conditions

Create one row per room or system:

```text
Area / observed condition / retained / removed / unknown / constraint / evidence / verification owner
```

Include structure, envelope, windows, doors, heating/cooling, ventilation, plumbing and drainage, electrical service, network, gas, waterproofing history, moisture, acoustics, fixed equipment, shafts, and building rules only where relevant.

Do not treat a clean finish as evidence that concealed work is sound. Use an opening-up survey or professional inspection when concealed conditions materially affect the plan.

## 4. Scope And Priorities

Classify every desired item:

- `must`: project fails without it;
- `should`: high value but can be redesigned;
- `could`: optional upgrade;
- `not-now`: deliberately excluded with dependencies recorded.

Add acceptance language. Replace “better storage” with a count or category of items, access frequency, preferred storage location, and maximum visible clutter.

## 5. Decision Ledger

Keep a dated table:

```text
ID / decision / status / options / chosen reason / evidence / owner / due date / affects / reversible
```

Statuses: `open`, `needs-measurement`, `needs-source`, `needs-professional`, `approved`, `superseded`.

## 6. Deliverable Map

Distinguish:

- concept: intent, adjacency, mood, priorities;
- measured design: coordinated dimensions and interfaces;
- technical design: system calculations and responsible professional approval;
- procurement: exact product, finish, quantity, lead time, warranty, approved sample;
- construction: issued information, hold points, approved changes;
- handover: tests, manuals, warranties, as-built records, defects, final account.

Never let a mood board or AI visualization masquerade as one of the latter stages.
