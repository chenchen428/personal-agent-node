# Translation Rules

## Before Translating

Identify target language, audience, locale, publication channel, desired register, and terms that must remain unchanged. For repeated specialist terms, create a compact source-to-target term map.

## While Translating

- Translate meaning at sentence and paragraph level.
- Keep code, commands, paths, URLs, identifiers, and frontmatter keys unchanged.
- Preserve Markdown hierarchy, links, image paths, and code fences.
- Retain hedging and uncertainty.
- Use established localized product or standards names; otherwise keep the original on first mention.
- Adapt punctuation and units only when the locale convention is clear and the value remains equivalent.

## Review

Check every number, negation, condition, heading, link label, and repeated term. Compare the opening, conclusion, and all warnings against the source. Flag ambiguous source wording rather than choosing a convenient interpretation silently.
