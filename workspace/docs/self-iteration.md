# 自迭代指南

自迭代的目标是让个人助手更贴合用户，而不是无边界地改写自己。每次迭代遵循下面的
闭环：

1. 从明确的用户目标或反复出现的问题开始，写清期望结果、影响范围和风险。
2. 先读 `AGENTS.md`、相关 registry、Skill 和 workflow，确认能力归属与权限。
3. 优先修改用户拥有的层：`apps/`、`skills/`、`workflows/`、自动化定义或说明文档；
   不修改 `../core/current`。
4. 为行为变化先补可复现用例，再做最小实现。涉及发布、发送、账号、删除或外部系统的
   操作，先给计划并等待用户确认。
5. 运行对应的 verify/status 和工作区检查，保留简洁、脱敏的结果。失败时回退本次变更，
   不掩盖问题。
6. 将真正影响用户的结果写入动态；不要把工具调用、日志或心跳当成动态。

迭代资产必须具备清晰名称、用途、输入输出、权限、验证方法和恢复方式。临时试验放在
可删除的草稿位置；经过验证且会重复使用的做法再沉淀为 Skill、workflow 或 App。

工作区不承担旧 CLI 的历史兼容。自动化和文档统一使用 `pa-cli`；运行时管理统一使用
`personal-agent`。
